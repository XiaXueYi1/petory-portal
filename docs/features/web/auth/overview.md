# Web Auth

## 目标

完成 Web 端最小登录闭环，确保登录页、登录请求、错误提示、成功跳转和基础登录态同步能够工作。

## 当前范围

- 新增 `/login` 页面
- 新增最小受保护根路由 `/`
- 对接后端 `/v1/auth/login` 与 `/v1/auth/profile`
- 保持界面简洁，沿用当前 `foundation` 的暖色主题方向
- 不扩展到完整后台页面、菜单树或动态路由落地

## 当前状态

- 已新增登录页与基础表单提交流程
- 已增加最小请求层和 auth API 封装
- 已增加登录态 store 与启动时 `profile` 同步
- 已增加登录成功后的临时首页占位
- `pnpm run lint` 待本轮校验结果回填

## 已知限制

- 当前默认按 Cookie / Session 方向对接，未处理 Token 模式
- 登录成功后只跳到临时首页占位，不包含真实业务菜单
- 未接入持久化菜单、权限组件和完整后台导航
