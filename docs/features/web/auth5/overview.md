# Web Auth5 Overview

## 目标

把 Web 端从 auth4 的过渡态收口为真实可用的登录体验，确保 mini 已注册用户可以使用 Web 正常登录，并在登录后进入真实首页承接页。

## 范围

- 登录页改成产品化页面，不保留调试说明风格
- 仍然使用 `phone + password`
- 仅对登录请求中的 `password` 做 `AES-256-GCM` 加密
- 登录成功后跳转首页
- 首页作为真实承接页，不再是调试说明页
- 保持现有请求层和路由结构，不大改整体技术栈

## 当前状态

- Web 登录请求仍走 `/v1/auth/login`
- 密码加密仍通过现有 shared crypto 实现
- 登录后仍会拉取 profile 并写入 auth store
- 当前需要的是页面体验收口，而不是重写登录协议

## 已知限制

- 本轮不扩展后台业务模块
- 本轮不重做路由架构
- 仍然依赖 server 侧登录契约可用
