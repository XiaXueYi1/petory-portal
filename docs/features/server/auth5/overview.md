# Server Auth5 Overview

## 目标

修复当前 `auth5` 登录闭环中的 Web 登录问题，确保 mini 端已注册用户能够直接使用 Web 登录。

## 本轮范围

- 只处理 `server` 侧登录链路
- 修正 mini 注册后 Web 登录失败的问题
- 确保登录契约和当前真实实现一致
- 不扩展 RBAC 和业务模块

## 当前修正点

- mini 登录仍然走 `phone + appCode(code)`
- mini 首次登录或重复登录时，服务端会同步维护 Web 可登录的密码身份
- 默认密码边界仍为 `123456`
- Web 登录仍按 `phone + password` 处理
- Web 登录密码可接受当前 auth5-fix 约定的 AES 加密字段，也兼容明文兜底

## 核心结论

- 当前问题不是 mini 用户没有账号，而是 Web 可登录的密码身份不够稳定
- `auth5` 的修复重点是让 mini 创建或同步后的用户，始终有一条可用于 Web 登录的 `WEB_PASSWORD` 身份
- 默认密码仍保持 `123456`，方便 Web 端直接登录验证

## 已知限制

- 当前没有独立的密码修改/重置流程
- 默认密码只用于当前开发联调边界
- 仍需确保 Web 端请求格式与服务端 AES 配置一致
