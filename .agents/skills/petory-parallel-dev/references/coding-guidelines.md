# 通用编码规范

## 核心原则

- 高内聚，低耦合
- 单一职责
- 先保证正确性，再谈抽象和优化
- 优先可读性
- 不为假设中的未来需求过度设计

## 函数设计

- 参数过多时改为配置对象
- 返回值语义明确
- 避免隐式副作用

## 命名

### 变量

- 使用完整语义
- 避免 `data1`、`temp`、`obj`

### 布尔值

- 使用 `is`、`has`、`can`、`should`

### 函数

- 用动词开头
- 明确表达动作

## 模块设计

- 一个模块只负责一个明确职责
- 模块之间通过清晰输入输出协作
- 不跨层直接依赖

## 抽象判断

适合抽象：

- 会被复用
- 逻辑已重复两次以上
- 抽出后语义更清晰

不适合抽象：

- 只用一次
- 抽出来名字更空泛
- 只是为了“看起来高级”

## 避免的做法

- 巨型函数
- 巨型文件
- 无边界的 `utils.ts`
- 深层嵌套判断
- 不透明副作用
