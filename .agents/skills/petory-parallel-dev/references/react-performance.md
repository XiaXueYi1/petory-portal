# React 性能建议

适用于 `petory-web/` 和需要 React 视图优化的场景。

## 基本原则

- 先保证正确性，再做性能优化
- 只优化真实热点
- 服务端状态和本地 UI 状态分离

## 避免重复渲染

- 不要把大对象和大数组直接内联传给子组件
- 需要稳定引用时再考虑 `useMemo`
- 只有在真实依赖场景里再考虑 `useCallback`

## `useCallback` 适用场景

- 传给被 memo 的子组件
- 作为副作用依赖项
- 高频渲染列表中的事件函数

## `useMemo` 适用场景

- 表格列定义
- 大量 options 转换
- 开销明显的过滤和映射

## 状态建议

### Zustand

- 使用精确 selector
- 不把服务端列表数据长期塞进 store

### Query

- query key 稳定且语义明确
- mutation 后精确失效缓存
- 避免同页重复请求同一数据

## 页面建议

- 大列表必须分页
- 重型弹窗内容按需挂载
- 页面拆成局部区域组件，避免一个 state 影响整页

## 避免的做法

- 机械式滥用 `React.memo`、`useMemo`、`useCallback`
- 没发现真实问题就提前优化
